# 交付 — throughline、STAR、時間預算、演練、speaker notes、demo 安全網

> 適用：內容定稿後的「怎麼講」。投影片只是輔助，**話由人講**——這層讓論證真的傳達出去。

## 1. Throughline（貫穿線）

**來源**：Chris Anderson《TED Talks: The Official TED Guide》。
- 整場由**一條連接所有段落的核心想法**串起（就是你的 governing thought 在口語層的延伸）。
- 一句話檢查：「如果觀眾只記得一件事，那是什麼？」每段都要能掛回這條線。

## 2. 開場與 STAR moment

- **60 秒 hook**：用驚人數字、一次 live 失敗的畫面、或一句挑釁的宣稱，**先賺到注意力再講脈絡**。
  不要用「先看一下 agenda」或自我介紹開場（見 anti-patterns.md）——**先給價值**。
  可仿的 hook 套路：**驚人數字**（「77% 企業說工具太多反成盲點」）、**嚇人後果**（「發現時帳單已破百萬美元」）、
  **流行文化標題**（藉熟梗製造記憶點）、**挑釁宣稱**。**上列數字僅為示意句型，引用前請查證真實來源。** 更多實戰招式見 field-patterns.md。
- **STAR moment（Something They'll Always Remember，Duarte）**：每場設計一個難忘時刻，五種選一：
  戲劇化演示、可複述的金句、震撼視覺、動人故事、驚人統計。

## 3. 時間預算與演練

- **時間預算**：每張內容頁約 1–2 分鐘；依時長反推頁數（Kawasaki：20 分鐘上限），砍到合身，**不要趕場**。
- **演練**：對著時鐘**大聲**跑整場（Duarte 為 18 分鐘 TED 練 ~30 小時）。沒整場跑過一遍＝沒完成。
- 練的時候抓：超時的段、講不順的轉場、需要的口頭橋接句。

## 4. Speaker notes

- 寫**你要說的話 + 計時/demo 提示**，**不是複製投影片**（marcusnelson：「為嘴寫，不為眼寫」）。
- 投影片放證據，notes 放台詞與提醒；兩者不重複。

## 5. Demo 安全網（資安/技術 talk 尤其關鍵）

- **錄一段備援影片**：live demo 失敗時直接播。
- **備一頁預期輸出 fallback**：網路/環境掛掉也能講完。
- **絕不把整場賭在 live demo 會動**——這是鐵則 8 的延伸。
- 把 demo 寫成有張力的故事（setup → 懸念 → 反轉 → 收割）見 field-patterns.md 第 3 招。

## 6. 收尾（recency）

- 最後一頁最被記得：放**最想留下的訊息 + 一個明確 CTA**（最多 3 個 takeaway）。
- 不要用「謝謝聆聽 / Q&A」當最後一張可見的內容——把 takeaway 放在它前面或同頁。
- CTA 變體：把結尾轉成**觀眾可自評的 ≤3 個問句**（「你的 X 做得到嗎？」），讓人帶著行動離場（見 field-patterns.md）。
